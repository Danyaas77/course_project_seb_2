# Security Policy

## Сообщение об уязвимостях

1. **Не создавайте публичные issue/PR.** Вместо этого используйте один из приватных каналов:
   - Email: `security@roommate-chores.local`
   - GitHub Security Advisory (Security → Advisories → Report a vulnerability)
   - Личный контакт преподавателя/ТА (Slack/Telegram) согласно инструкциям курса
2. Включите:
   - краткое описание уязвимости и потенциального влияния;
   - шаги воспроизведения и ожидаемый/фактический результат;
   - информацию об окружении (commit, ОС, используемые команды);
   - предложенный фикс (опционально).
3. Мы подтвердим получение репорта в течение **2 рабочих дней**, обновим статус решения не реже одного раза в неделю и согласуем дату раскрытия.

## Ожидания от исследователей

- Действуйте добросовестно, не нарушайте приватность соседей по квартире и не пытайтесь извлечь данные, к которым у вас нет прав.
- Используйте только ваши собственные аккаунты/данные тестирования.
- Согласовывайте любые проверки производительности перед запуском.

## Обработка чувствительных данных

- **Не** добавляйте секреты и API-ключи в репозиторий. Настраивайте переменные окружения (`APP_API_KEY`, строки подключения) локально либо через секреты CI/CD.
- Используйте только синтетические данные без ПДн и платёжной информации.

## Поддерживаемые версии

- Поддерживается только ветка `main`. Для других веток фикс уязвимостей не гарантируется.

## Связанная документация

- Формат ошибок описан в `README.md` и реализован в `app/main.py`.
- CI-пайплайн (`.github/workflows/ci.yml`) выполняет линтеры и тесты, что помогает предотвратить регрессии безопасности.
